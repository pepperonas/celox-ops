## Project Overview

celox ops is a full-stack business management webapp for freelancers/IT consultants. German-language UI. **Multi-user with isolated workspaces** (per-user `owner_id` data isolation; admin-managed accounts, no public registration; JWT auth). Manages customers, orders, contracts, invoices (with PDF generation), leads, time tracking, expenses, the **Rainmaker** acquisition-activation module, and integrates with the Claude Token Tracker for AI usage transparency. The frontend uses a **Material Design 3 Expressive** dark theme.

Kein Auto-Signup (403 fأ¼r unverknأ¼pfte Konten); TOTP wird bei Google-Login bewusst nicht zusأ¤tzlich verlangt. Leere `GOOGLE_CLIENT_ID` â‡’ 503 + Button ausgeblendet (Login.tsx lأ¤dt das GIS-Script nur, wenn `GET /api/auth/info` eine `google_client_id` liefert â€” Laufzeit-Konfig, kein Frontend-Rebuild bei أ„nderung). - **Multi-tenancy invariants (important)**: (1) **Public endpoints and background jobs** run with the ContextVar UNSET â†’ queries are UNSCOPED (global). Any public route that returns owned data MUST resolve the user and `current_owner_id.set(user.id)` itself (see `routers/ical.py`, which authenticates via per-user `users.ical_token`). The cron (`run_cron`) sets the ContextVar per active user for the Rainmaker reminder; `check_overdue_invoices` is intentionally global. `backup.py` export sets it to `None` for a true full dump. (2) **Invoice numbers are unique PER OWNER** (`uq_invoice_owner_number`), not globally â€” `generate_invoice_number` is owner-scoped so each user has their own sequence. It takes a per-owner `pg_advisory_xact_lock` (held to commit) so concurrent creates (double-submit, or cron vs. the manual generate endpoint) serialize instead of colliding on the unique constraint; `flush_new_invoice` is a SAVEPOINT-based retry safety net. (3) When accepting an FK id (customer/order/contract/â€¦) in a request body, validate it with a scoped `select` (returns None if it belongs to another tenant) before assigning â€” `with_loader_criteria` only hides rows on SELECT, it does not validate inserts.
- **Multi-tenancy / data isolation**: `backend/app/tenancy.py` â€” every owned model uses `OwnedMixin` (`owner_id` FKâ†’users). A request-scoped `current_owner_id` ContextVar (set in `get_current_user`) drives two SQLAlchemy session events: `do_orm_execute` auto-filters every ORM SELECT (incl. column aggregates) to the owner via `with_loader_criteria`, and `before_flush` auto-stamps `owner_id` on new owned objects. Unset ContextVar (login, lifespan bootstrap, cron) â‡’ unscoped/global. **So routers generally need NO manual owner filtering** â€” but caches must be per-owner (e.g. dashboard `_stats_cache` is keyed by `owner_id`). `users`/`audit_log`/`document_templates` are global (not owned). Owned-model list is registered in `main.py` (`set_owned_models([...])` + `install_tenancy_events()`).
- **Pattern**: Each module has model â†’ schema â†’ router. All routers registered in main.py after app creation (to avoid circular imports).
- **Models inherit from `Base`** defined in `models/customer.py`. New models MUST import Base from there.
- **All list endpoints return paginated**: `{items: [], total, page, page_size, pages}` â€” NOT plain arrays. Exception: `GET /api/attachments` returns a plain list.
- **UUIDs**: All primary keys are UUID (asyncpg uses its own UUID type â€” serialize with `str()` for JSON).
- **Enum values**: Stay ASCII in code (`ueberfaellig`, `gekuendigt`, `halbjaehrlich`). German umlauts only in user-facing text/labels.
- **PDF generation**: WeasyPrint + Jinja2 templates in `backend/app/templates/`. Signature loaded as base64. Footer uses `@page @bottom-center` (not position:fixed).
- **Cron**: Background asyncio task in lifespan, hourly. Globally: marks overdue invoices (`check_overdue_invoices`). Then **per active user** (ContextVar scoped): Rainmaker daily reminder + **auto-recurring invoices** (`services/invoice_service.generate_due_recurring` â€” creates draft invoices for due contract billings; idempotent via `last_invoiced_date`, same logic as `POST /api/invoices/generate-recurring`).
- **Backup export** (`routers/backup.py`): Auto-discovers all tables via `Base.registry.mappers` â€” no manual list needed for new tables.
