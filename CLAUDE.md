## Project Overview

celox ops is a full-stack business management webapp for freelancers/IT consultants. German-language UI. **Multi-user with isolated workspaces** (per-user `owner_id` data isolation; admin-managed accounts, no public registration; JWT auth). Manages customers, orders, contracts, invoices (with PDF generation), leads, time tracking, expenses, the **Rainmaker** acquisition-activation module, and integrates with the Claude Token Tracker for AI usage transparency. The frontend uses a **Material Design 3 Expressive** dark theme.

## Architecture

### Backend (Python 3.12, FastAPI)
- **Entry point**: `backend/app/main.py` βÄî FastAPI app with lifespan (creates tables + starts hourly cron)
- **Config**: `backend/app/config.py` βÄî Pydantic Settings, all from `.env`
- **Database**: `backend/app/database.py` βÄî async SQLAlchemy 2.0 with asyncpg, connection pooling
- **Auth**: `backend/app/auth.py` βÄî JWT (python-jose), DB-backed `users` table, OAuth2PasswordBearer. `get_current_user` returns the `User` and sets the `current_owner_id` ContextVar (see Multi-tenancy). `require_admin` gates admin endpoints. `.env` admin is bootstrapped into a real DB user on first start. **Google-Login ("Sign in with Google")**: `POST /api/auth/google` verifiziert das GIS-ID-Token (google-auth: Signatur/exp/aud gegen `GOOGLE_CLIENT_ID` aus .env; `validate_google_claims` prΟΦft zusΟΛtzlich iss/email_verified βÄî DB-frei getestet) und meldet den Benutzer an, dessen `users.google_email` (unique, case-insensitiv verglichen, via manuellem `ALTER` migriert) zur E-Mail passt. Kein Auto-Signup (403 fΟΦr unverknΟΦpfte Konten); TOTP wird bei Google-Login bewusst nicht zusΟΛtzlich verlangt. Leere `GOOGLE_CLIENT_ID` β΅£ 503 + Button ausgeblendet (Login.tsx lΟΛdt das GIS-Script nur, wenn `GET /api/auth/info` eine `google_client_id` liefert βÄî Laufzeit-Konfig, kein Frontend-Rebuild bei Ο³nderung). JWKS-Fetch lΟΛuft via `asyncio.to_thread`; Rate-Limit 10/min/IP. Deps: `google-auth` + `requests`. VerknΟΦpfung pflegt der Admin ΟΦber `PATCH /api/users/{id}` (`google_email`; "" entfernt sie).
- **Multi-tenancy invariants (important)**: (1) **Public endpoints and background jobs** run with the ContextVar UNSET βÜ£ queries are UNSCOPED (global). Any public route that returns owned data MUST resolve the user and `current_owner_id.set(user.id)` itself (see `routers/ical.py`, which authenticates via per-user `users.ical_token`). The cron (`run_cron`) sets the ContextVar per active user for the Rainmaker reminder; `check_overdue_invoices` is intentionally global. `backup.py` export sets it to `None` for a true full dump. (2) **Invoice numbers are unique PER OWNER** (`uq_invoice_owner_number`), not globally βÄî `generate_invoice_number` is owner-scoped so each user has their own sequence. It takes a per-owner `pg_advisory_xact_lock` (held to commit) so concurrent creates (double-submit, or cron vs. the manual generate endpoint) serialize instead of colliding on the unique constraint; `flush_new_invoice` is a SAVEPOINT-based retry safety net. (3) When accepting an FK id (customer/order/contract/βÄΠ) in a request body, validate it with a scoped `select` (returns None if it belongs to another tenant) before assigning βÄî `with_loader_criteria` only hides rows on SELECT, it does not validate inserts.
- **Multi-tenancy / data isolation**: `backend/app/tenancy.py` βÄî every owned model uses `OwnedMixin` (`owner_id` FKβÜ£users). A request-scoped `current_owner_id` ContextVar (set in `get_current_user`) drives two SQLAlchemy session events: `do_orm_execute` auto-filters every ORM SELECT (incl. column aggregates) to the owner via `with_loader_criteria`, and `before_flush` auto-stamps `owner_id` on new owned objects. Unset ContextVar (login, lifespan bootstrap, cron) β΅£ unscoped/global. **So routers generally need NO manual owner filtering** βÄî but caches must be per-owner (e.g. dashboard `_stats_cache` is keyed by `owner_id`). `users`/`audit_log`/`document_templates` are global (not owned). Owned-model list is registered in `main.py` (`set_owned_models([...])` + `install_tenancy_events()`).
- **Pattern**: Each module has model βÜ£ schema βÜ£ router. All routers registered in main.py after app creation (to avoid circular imports).
- **Models inherit from `Base`** defined in `models/customer.py`. New models MUST import Base from there.
- **All list endpoints return paginated**: `{items: [], total, page, page_size, pages}` βÄî NOT plain arrays. Exception: `GET /api/attachments` returns a plain list.
- **UUIDs**: All primary keys are UUID (asyncpg uses its own UUID type βÄî serialize with `str()` for JSON).
- **Enum values**: Stay ASCII in code (`ueberfaellig`, `gekuendigt`, `halbjaehrlich`). German umlauts only in user-facing text/labels.
- **PDF generation**: WeasyPrint + Jinja2 templates in `backend/app/templates/`. Signature loaded as base64. Footer uses `@page @bottom-center` (not position:fixed).
- **Cron**: Background asyncio task in lifespan, hourly. Globally: marks overdue invoices (`check_overdue_invoices`). Then **per active user** (ContextVar scoped): Rainmaker daily reminder + **auto-recurring invoices** (`services/invoice_service.generate_due_recurring` βÄî creates draft invoices for due contract billings; idempotent via `last_invoiced_date`, same logic as `POST /api/invoices/generate-recurring`).
- **Backup export** (`routers/backup.py`): Auto-discovers all tables via `Base.registry.mappers` βÄî no manual list needed for new tables.

### Legal-Compliance-Tracking (Rechtsdokumente)
Stellt sicher, dass jeder Kunde die Pflicht-Rechtsdokumente (Default **AGB + AVV**) unterschrieben vorliegen hat. Router `routers/compliance.py` (`/api/compliance/*`): `GET /overview` (alle Kunden Ο½ Pflichtdocs, LΟΦcken zuerst), `GET /customer/{id}`, `POST /mark` (manuell ab-/anhaken), `POST /upload` (unterschriebenes PDF βÜ£ legt einen `attachments`-Eintrag an **und** den Compliance-Record an), `PUT /templates/{id}/required` (Pflicht-Flag toggeln).

- **Frontend**: Nav-Seite **Rechtsdokumente** (`/rechtsdokumente`, `pages/Compliance.tsx`) mit Summary + Pro-Kunde-Karten (PDF erstellen / Hochladen / Abhaken) + Pflicht-Konfiguration.
